# 技术报告

## 1. 项目目标

本项目面向赛道 C: Edge AI，目标是构建一个可离线运行、可连接真实硬件、具备多模态感知能力的智能盆栽系统。系统需要同时解决以下真实痛点：

- 普通智能花盆只能给出数值，缺乏自然交互与情感陪伴。
- 传统云端方案依赖网络，时延高且存在隐私风险。
- 家庭植物养护往往缺少可持续提醒与低门槛交互入口。

因此，本方案选择将 Gemma 4 部署到端侧设备配套 App 中，让植物识别、对话和记忆总结尽可能在本地完成。

## 2. 模型选型理由

### 2.1 为什么选择 Gemma 4

Gemma 4 具备以下优势：

- 适合做端侧落地，便于在移动设备上结合量化模型运行。
- 支持高质量指令跟随，适合自然对话与角色化表达。
- 具备多模态扩展能力，适合植物图像识别场景。
- 生态成熟，便于通过 GGUF 量化与本地推理框架接入。

### 2.2 为什么选择 gemma-4-E2B-it-GGUF

赛道 C 更强调离线部署与硬件适配，因此本项目当前提交版本选择 `gemma-4-E2B-it-GGUF`，原因如下：

- 该版本已经在本项目现有端侧链路中完成了实际接入与验证。
- 能兼容当前的 GGUF 本地加载方式，便于评委按提交文档进行复现。
- 能支撑聊天、多模态识别、记忆压缩等核心任务。


如果后续硬件条件允许，也可继续尝试 Gemma 4 的其他规格，但本提交包默认按 `gemma-4-E2B-it-GGUF` 的实际运行版本进行说明。

## 3. 架构设计

整体架构分为两层：

### 3.1 App 端

- Flutter 负责 UI、配置管理、对话、拍照、状态展示。
- `llamadart` 负责本地 GGUF 模型加载与流式推理。
- 多模态识别将植物图片与文本提示共同输入模型。
- `SharedPreferences` 保存本地配置与记忆摘要。

### 3.2 硬件端

- ESP32-S3 采集土壤湿度、温度、PIR、摄像头数据。
- 本地 WebServer 提供状态与控制接口。
- 屏幕负责显示植物情绪与传感器状态。
- Wi-Fi 仅用于局域网通信与可选 NTP 校时，不依赖公网推理。

## 4. Gemma 4 特性利用

### 4.1 多模态处理

项目核心亮点之一是植物识别：

- 用户拍摄植物图片
- App 将图片与文本提示共同送入本地模型
- Gemma 4 输出植物名称、标签、人格签名与角色风格
- 系统将识别结果转化为后续聊天人格

这部分对应赛题对多模态能力的要求，也增强了智能盆栽与真实世界的连接。

### 4.2 端侧部署

本项目将模型运行放在移动端而非云端：

- 降低网络依赖
- 提高交互私密性
- 缩短响应链路
- 提升比赛现场可控性


### 4.3 记忆压缩

项目使用本地模型对历史互动进行总结：

- 保留关键信息
- 降低长期对话的上下文压力
- 保持植物人格连续性

这使系统不仅能“回答”，还能“记住”。

## 5. 关键工程实现

### 5.1 软件侧

- 本地模型加载与推理：`APP/lib/services/local_llama_service.dart`
- 多模态植物识别：`APP/lib/services/plant_recognition_service.dart`
- 聊天与上下文构造：`APP/lib/providers/chat_provider.dart`
- 记忆总结：`APP/lib/providers/memory_provider.dart`
- 硬件通信：`APP/lib/services/planter_service.dart`

### 5.2 硬件侧

- 固件入口：`firmware/src/main.cpp`
- 硬件联网：`firmware/src/wifi_time.cpp`
- Web API：`firmware/src/web_server.cpp`
- 摄像头：`firmware/src/camera_manager.cpp`
- 传感器与情绪控制：`firmware/src/plant_controller.cpp`

## 6. 数据合规与隐私保护

虽然本项目主赛道为 Edge AI，但依然对隐私进行了处理：

- 不在提交代码中写死任何 Wi-Fi 密码
- 不写死开发者本地串口、构建路径、局域网地址
- 默认不写入 API Key
- 尽量在端侧本地完成推理
- 图片数据不需要上传到云端即可完成识别

固件通过 `wifi_credentials.example.h` 提供模板，评委只需本地复制并填写，不会将现场网络信息提交到仓库。

## 7. 可复现性设计

为了方便评委复现，本提交包已进行如下整理：

- 软硬件代码已拆分到独立提交目录
- README 中给出环境要求与完整复现步骤
- 固件联网配置改为外部模板文件
- 删除作者本机相关路径与端口
- 配置页默认值改为空白或通用占位

## 8. 总结

本项目的核心价值在于把 Gemma 4 从“聊天模型”扩展为“可感知、可联动、可离线运行的植物陪伴系统”：

- 它能看见植物
- 它能理解状态
- 它能记住互动
- 它能通过真实硬件反馈情绪

这使其符合赛道 C 对 Edge AI、硬件适配、离线能力和实际应用价值的要求。
