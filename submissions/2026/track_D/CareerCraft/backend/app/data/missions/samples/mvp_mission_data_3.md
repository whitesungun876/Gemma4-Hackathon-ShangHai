# 跨部门 KPI 指标定义草稿

## 背景
销售部、市场部、产品部目前各自报数：销售看 GMV，市场看曝光-点击转化，产品看 DAU。
本草稿目标是收敛出**统一的核心 KPI 三件套** 供管理层周会复盘。

## 候选 KPI

### 1. 付费转化率 (Paid Conversion Rate)
- 公式：`paid_conversion_rate = paid_users / dau`
- 颗粒度：按日 / 按渠道 / 按客群
- 边界：仅统计自然日内首次完成支付的用户，退款不扣减

### 2. 客单价 (ARPPU)
- 公式：`arppu = gmv / paid_users`
- 颗粒度：按日 / 按品类
- 边界：使用支付完成时刻的实付金额（含优惠扣减后），不使用挂单金额

### 3. 次日留存率 (D1 Retention)
- 公式：`d1_retention = retained_users_d1 / new_users_d0`
- 颗粒度：按注册渠道 / 按首日行为分桶
- 边界：D1 指注册次日 00:00–23:59 内有任意一次有效行为（页面浏览除外）

## 待对齐冲突点
- 销售希望把"加购成功"算入支付转化（产品反对：与 GMV 口径不一致）
- 市场希望把"邀新带来的二次下单"计入次日留存 D1（产品反对：会引入幸存者偏差）
- 产品希望把推送点击算入 DAU（市场反对：会虚高活跃数字）

## 仪表盘布局建议
- 顶部：3 个核心数值卡片（付费转化、ARPPU、D1）
- 中部：7 日趋势折线 + 同比基线
- 底部：按渠道 / 客群下钻表，可点击进入明细
