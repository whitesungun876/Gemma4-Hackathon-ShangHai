# 签到功能 API 设计模板（待澄清）

## 背景
PM Amy 提议增加每日签到功能以提升 DAU。需求文档目前为空，仅有一句话："用户每天可以签到一次，连续签到给奖励"。

## 已知未澄清点
- "每天一次" 指自然日（00:00 本地时区）还是 24 小时滚动窗口？
- 时区：服务器时区还是用户本地时区？跨时区出差账号怎么处理？
- 连续签到中断的判定：错过一天清零，还是允许"补签"？补签是否消耗道具？
- 奖励机制：是固定积分、阶梯奖励，还是连续 N 天解锁特殊礼包？
- 积分的来源/上限/是否可与其他活动叠加？
- 失败场景：网络抖动重复点击是否算多次？需要幂等吗？
