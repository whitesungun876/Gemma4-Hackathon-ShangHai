# 贡献指南

欢迎来到 CareerCraft 项目！以下是参与开发的流程。

## 开发流程
1.  **Fork 本仓库**到你的账户。
2.  **创建特性分支** (`git checkout -b feature/AmazingFeature`)。
3.  **提交你的更改** (`git commit -m 'Add some AmazingFeature'`)。
4.  **推送到分支** (`git push origin feature/AmazingFeature`)。
5.  **开启一个 Pull Request**。

## 角色与主要职责 (概要)
项目遵循前后端分离、模块化开发的协作模式：
- **产品/主策**：负责产品定义、内容生产、核心玩法与世界观设计。
- **后端/AI架构师**：负责整体架构、Gemma 4 Agent核心逻辑、API设计。
- **全栈/RAG工程师**：负责RAG知识库系统搭建、业务逻辑辅助开发。
- **前端工程师**：负责像素风Web界面实现、交互逻辑与前后端联调。

## 代码与提交规范
- 请遵循现有代码风格。
- 提交信息应清晰描述更改内容。
- 确保你的代码通过了基本的代码检查。

## 沟通
- 主要开发讨论在项目的 [Issues](https://github.com/your-org/careercraft/issues) 和 [Pull Requests](https://github.com/your-org/careercraft/pulls) 中进行。
- 对于更详细的内部开发计划与排期，请参考内部项目管理文档。

**感谢您的贡献！**
