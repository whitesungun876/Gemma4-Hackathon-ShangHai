# 数据叙事框架：如何从数字到故事

## TL;DR

- **核心规则**：用"情境–冲突–解决方案"（SCQA）三段式包装分析，让业务方能听懂、记住、采取行动。
- **反例**：写一份"本月各品类销售额是 X、Y、Z…"的流水账，业务方读完不知道该做什么。
- **在本任务里怎么用**：按 §2 的模板组织 `insight_report.md`，每个洞察都要带具体行动建议。

## 1. 经典叙事结构：情境–冲突–解决方案 (SCQA)

- **情境 (Situation)**：交代背景。"我们上个月整体销售额 X 元，各品类表现如下…"
- **冲突 (Complication)**：发现问题或机会。"但是，我们发现增长不均衡，数码配件增长迅猛，而家居品类停滞。"
- **解决方案 (Resolution)**：提出你的建议。"因此，我们建议下月将营销预算向数码配件倾斜 20%，并策划'配件焕新'专题活动。"

## 2. 针对"洞察炼金术师"任务的应用

`insight_report.md` 可以这样组织：

### 核心数据摘要（情境）

- 本月总销售额：2,165,000 元，环比增长 12.5%。
- 销售额 Top 3 品类：数码配件 (30.5%)、电子产品 (27.7%)、美妆 (16.6%)。

### 关键业务洞察（冲突 / 机会）

- **增长引擎**：数码配件不仅是销售额第一品类（占比 30.5%），且环比增速高达 65%，是当前最强劲的增长动力。其高增长可能受益于近期手机新品发布带来的配件需求。
- **流量基石**：服装品类销量占比最高 (28.7%)，但平均单价最低 (50 元)，表明它是吸引用户、拉动流量的关键，但利润贡献有限。
- **隐忧品类**：家居品类销售额占比 11.1%，但环比零增长，需关注其竞争力是否下降。

### 具体行动建议（解决方案）

基于洞察 1，建议：

- **策略**：立即策划"春季配件焕新"促销活动，针对数码配件品类，捆绑高单价电子产品进行联合优惠（如买手机加 9.9 元换购保护壳）。
- **目标**：提升整体客单价 10%，并进一步巩固数码配件的增长势头。
- **理由**：利用明星品类的拉力，带动高利润品类的销售，实现业绩最大化。

## 3. 好故事的要点

- **一句话核心观点**：放在开头。
- **数据支撑**：每个观点后紧跟关键数据。
- **关联业务**：始终回答"这对我（业务方）意味着什么？"
- **建议具体**：谁、在什么时间、做什么、预期什么结果。

> 你不是在汇报数据，你是在用数据论证一个商业决策。

## 4. insight_report.md 推荐结构

- 核心结论：先给一句话结论。
- 数据证据：列出支撑结论的关键指标。
- 业务解释：说明这些指标为什么重要。
- 行动建议：给出具体动作，而不是抽象方向。
- 风险与验证：说明建议可能的风险，以及如何验证效果。

## 5. 从数字到建议的句式

- "因为 A 指标表现为 X，所以说明 B 业务现象，建议采取 C 动作。"
- "虽然 A 品类增长快，但基数较小，因此建议先小规模测试。"
- "该渠道转化高但成本也高，建议优化投放素材，而不是直接扩大预算。"

## 常见误区

- 不要罗列所有指标，只保留能支撑故事主线的关键数字。
- 不要给"加大投入""优化体验"这种空建议，必须落到可执行动作。
- 不要省略风险与验证，让业务方知道这个建议怎么被证伪。
