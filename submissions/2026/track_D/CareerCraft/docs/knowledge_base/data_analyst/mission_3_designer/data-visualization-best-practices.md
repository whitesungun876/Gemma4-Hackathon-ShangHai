# 数据可视化最佳实践：一图胜千言

## TL;DR

- **核心规则**：先问"我要回答什么业务问题"，再按 §1 表格选图，让读者 5 秒内读出结论。
- **反例**：用 3D 饼图展示渠道占比，或在折线图里塞 8 个指标。
- **在本任务里怎么用**：按 §2 为漏斗、渠道对比、趋势分别选图，按 §4 描述写进 `dashboard_prototype.md`。

## 1. 图表类型选择指南

| 你想展示什么？ | 推荐图表 | 示例（任务相关） |
| :--- | :--- | :--- |
| 部分与整体关系 | 饼图、环形图、堆叠柱状图 | 各渠道付费用户占比 |
| 比较不同类别 | 柱状图、条形图 | 不同渠道的获客成本 (CAC) 对比 |
| 数据随时间变化 | 折线图、面积图 | 每日付费用户数趋势（7 天） |
| 分布情况 | 直方图、箱线图 | 用户订单金额分布 |
| 两个变量关系 | 散点图 | 广告花费 vs. 带来的收入 |
| 流程或漏斗 | 漏斗图 | 用户从曝光到付费的转化漏斗 |

## 2. 为"看板设计师"任务选择图表

基于数据和沟通结果：

- **漏斗转化图 (Funnel Chart)**：必须包含。直观展示用户流失。工具：Plotly、Matplotlib + 自定义。
- **分组柱状图 (Grouped Bar Chart)**：对比不同渠道在点击、注册、付费各环节的绝对数量。
- **折线图 (Line Chart)**：展示核心指标（如付费用户数）随时间（过去 7 天）的趋势。

## 3. 设计原则：让图表清晰易懂

- **标题明确**：不要用"销售数据"，用"各渠道用户获取漏斗转化对比 (2026-04-20 至 22)"。
- **标注关键信息**：柱子上标注具体数值，漏斗环节标注转化率。
- **颜色使用**：同一渠道在不同图表中用相同颜色。
- **避免"图表垃圾"**：去除不必要的 3D 效果、阴影、花哨背景。
- **纵轴从 0 开始**：避免误导性缩放（除非有特殊原因）。

## 4. 动手制作原型

在 `dashboard_prototype.md` 中可以这样描述：

### 图表 1：用户获取漏斗（整体）

- **类型**：漏斗图
- **数据**：
  - 环节：曝光 → 点击 → 注册 → 付费
  - 数值：`[200000, 10000, 1900, 190]`（计算自三日总和）
  - 转化率：`[5.0%, 19.0%, 10.0%]`（标注在环节间）
- **洞察**：注册到付费环节流失最严重（仅 10% 转化），是优化重点。

### 图表 2：分渠道付费用户与 CAC 对比

- **类型**：分组柱状图
- **X 轴**：渠道 (Search, Social, Direct)
- **Y 轴（左）**：付费用户数（柱状）
- **Y 轴（右）**：获客成本 CAC（折线或点）
- **洞察**：Search 渠道带来最多付费用户，但 CAC 也最高；Direct 渠道 CAC 最低，效率最高。

## 5. 看板图表选择检查表

选择图表时先问：

- 我要比较大小，还是展示趋势？
- 我要展示构成，还是展示转化路径？
- 读者最关心成本、收入、转化还是留存？
- 图表是否能在 5 秒内读出结论？

用户获取漏斗适合用漏斗图展示环节流失，用柱状图比较渠道，用折线图展示时间趋势。

## 常见误区

- 不要在一个图里塞太多指标。销售、市场、产品关注点不同，可以用分区布局解决。
- 不要只画漂亮图表。每个图表都要回答一个业务问题。
- 不要省略单位和口径，例如转化率是点击到注册，还是曝光到注册。
