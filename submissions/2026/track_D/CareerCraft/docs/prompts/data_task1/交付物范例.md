# 任务一「数据侦探」交付物范例

本文档定义了任务“数据侦探”中，用户需提交的关键交付物的“优秀”与“及格”标准范例，用于训练AI（Gemma 4）进行准确评估。

## 1. 数据清洗规则文档 (data_cleaning_rules.md)
**优秀范例 (对应高质量反馈)**

文档标题：促销活动数据清洗规则 v1.0

概述

本规则旨在处理promo_data_raw.csv中的质量问题，为后续转化率分析提供干净、可靠的数据基准。

### 1. 缺失值处理

customer_id(主键)：缺失3条。处理：整行删除。理由：无法关联用户行为，无分析价值。

purchase_amount(核心指标)：缺失5条。处理：采用同一city& product_category下的中位数进行填充。理由：避免全局均值带来的分布扭曲，基于同类用户行为进行估算更合理。

campaign_channel(维度)：缺失2条。处理：根据user_agent字段推断为Social Media，并在该字段末尾添加标记[INFERRED]。理由：保留数据，但明确标记不确定性。

### 2. 异常值与无效值处理

purchase_amount(范围)：设定业务合理范围为$0.01 - $1000。发现2条记录为$2500。处理：缩尾处理(Winsorization)，将其调整为第99百分位数的值($850)。理由：保留极端高消费用户，但避免其扭曲统计结果。

age(逻辑)：设定合理范围为18-100。发现1条记录为150。处理：整行删除。理由：明显为无效输入。

### 3. 格式与一致性处理

event_timestamp：统一格式为YYYY-MM-DD HH:MM:SS，并转换为UTC时区。

product_category：建立映射表，将Electornics, elec等7种变体统一为Electronics。

### 4. 逻辑一致性校验

规则1：若event_type为purchase，则purchase_amount必须>0。发现1条冲突记录，将其event_type修正为refund。

规则2：registration_date必须 ≤ 同一用户的首次event_timestamp。发现5条违规，以首次page_view时间修正registration_date。

### 5. 衍生字段

新增is_weekend：布尔值，基于event_timestamp判断是否为周末。

新增customer_segment：基于历史购买频率（临时计算），分为新客/回头客/沉睡客。

**及格范例 (对应勉强通过/需要改进的反馈)**

## 文档标题：清洗规则

### 1. 处理空值：

customer_id为空的，直接删除。

purchase_amount为空的，填0。

其他字段为空的，填“Unknown”。

### 2. 处理错误值：

purchase_amount大于1000的，改成1000。

age不在18-100岁之间的，删除。

### 3. 统一格式：

product_category里的“Electornics”改成“Electronics”。

## 2. 清洗后数据结果描述
**优秀描述**

“应用上述规则后，原始10，000行数据剩余9，989行。主要变动：删除无效行（3条customer_id缺失 + 1条异常age），修正并填充5条purchase_amount，统一了产品类别的7种变体，修复了5条用户时间线逻辑。核心字段 (customer_id, purchase_amount, event_timestamp) 现已100%完整且合乎逻辑，数据集已可用于可靠的转化率分析。”

**及格描述**

“删掉了一些明显有问题的行，比如ID没有的、年龄不对的。把空的价格填成了0。现在数据看起来整齐多了，应该可以算转化率了。”

## 3. 转化率计算逻辑与结果
**优秀范例 (结构化、有下钻)**

报告标题：转化率分析报告

### 1. 核心定义

目标转化事件：event_type= ‘purchase’

总触达用户数：在活动期内，有过ad_view或email_open事件的去重用户数。

### 2. 整体转化率

总触达用户数：8,542

转化用户数：1,234

整体转化率：(1,234 / 8,542) * 100% = 14.45%

### 3. 多维下钻分析

| 维度 | 细分 | 触达用户数 | 转化用户数 | **转化率** | 洞察 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **渠道** | `Email` | 3,200 | 560 | **17.5%** | **表现最佳渠道** |
| | `Social Media` | 4,100 | 520 | 12.7% | 流量大但转化一般 |
| | `Search Engine` | 1,242 | 154 | 12.4% | 表现有提升空间 |
| **产品** | `Electronics` | 2,100 | 382 | **18.2%** | **高潜力品类** |
| | `Home & Kitchen` | 3,050 | 460 | 15.1% | 表现稳定 |
| | `Apparel` | 3,392 | 332 | 9.8% | **需重点关注** |
4. 核心结论
Email渠道效率最高，转化率显著优于其他渠道。

Electronics品类的转化率领先，可能与促销力度或需求有关。

Apparel品类转化率偏低，建议检查商品页面或促销策略。

及格范例 (仅基础计算)

“转化率 = 购买人数 / 看到广告的人数。购买的有1234人，看到广告的有8542人，除了一下大约是14.5%。邮件渠道的转化率好像高一点。”
