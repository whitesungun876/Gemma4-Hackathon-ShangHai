## 1. 修复后的源代码 (`UserService.java`)

### 优秀范例 (对应高质量反馈)
java

import org.apache.commons.lang3.StringUtils;

import java.util.HashMap;

import java.util.Map;

/**

用户服务类，处理用户注册与登录。

修复了用户名空格导致的登录问题。

*/

public class UserService {

private Map<String, String> userDatabase = new HashMap<>(); // key: username (trimmed), value: password

/**

用户注册

@param username 用户名（前后空格将被自动去除）

@param password 密码

@return 注册成功返回true，用户名已存在返回false

*/

public boolean register(String username, String password) {

if (StringUtils.isBlank(username) || StringUtils.isBlank(password)) {

return false;

}

String trimmedUsername = username.trim();

if (userDatabase.containsKey(trimmedUsername)) {

return false; // 用户已存在

}

userDatabase.put(trimmedUsername, password);

return true;

}

/**

用户登录

@param username 用户名

@param password 密码

@return 用户名密码匹配返回true，否则返回false

*/

public boolean login(String username, String password) {

if (username == null || password == null) {

return false;

}

String trimmedUsername = username.trim();

String storedPassword = userDatabase.get(trimmedUsername);

return storedPassword != null && storedPassword.equals(password);

}

}

### 及格范例 (对应勉强通过/需要改进的反馈)

java

import java.util.HashMap;

import java.util.Map;

public class UserService {

private Map<String, String> userDatabase = new HashMap<>();

public boolean register(String username, String password) {
    // 只做了null检查，没做trim
    if (username == null || password == null) return false;
    if (userDatabase.containsKey(username)) {
        return false;
    }
    userDatabase.put(username, password); // Key 没 trim
    return true;
}

public boolean login(String username, String password) {
    if (username == null) return false;
    // 登录时 trim 了，但注册时没 trim，导致不一致
    String trimmedUsername = username.trim();
    String storedPassword = userDatabase.get(trimmedUsername);
    return storedPassword != null && storedPassword.equals(password);
}

}

## 2. 单元测试文件 (`UserServiceTest.java`)

### 优秀范例

java

import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.*;

public class UserServiceTest {

@Test
public void testRegisterAndLoginWithSpaceInUsername_ShouldSuccess() {
    UserService service = new UserService();
    String usernameWithSpaces = "  alice  ";
    String password = "pass123";

    // 1. 注册应成功
    boolean registerResult = service.register(usernameWithSpaces, password);
    assertTrue(registerResult, "注册应该成功");

    // 2. 使用带前后空格的相同用户名登录应成功
    boolean loginResultWithSpaces = service.login("  alice  ", password);
    assertTrue(loginResultWithSpaces, "使用带空格的用户名登录应成功");

    // 3. 使用trim后的用户名登录也应成功
    boolean loginResultTrimmed = service.login("alice", password);
    assertTrue(loginResultTrimmed, "使用trim后的用户名登录应成功");

    // 4. 错误密码应失败
    boolean loginWrongPass = service.login("alice", "wrong");
    assertFalse(loginWrongPass, "错误密码应登录失败");
}

@Test
public void testRegisterDuplicateUser_ShouldFail() {
    UserService service = new UserService();
    assertTrue(service.register("bob", "pass"));
    assertFalse(service.register(" bob ", "pass2"), "用户名trim后重复，应注册失败");
}

}

## 3. Bug根因总结 (文本)

### 优秀总结
“Bug的根本原因是 `register` 和 `login` 方法对用户名的处理不一致。`register` 方法将原始用户名（可能包含前后空格）直接作为Key存入Map，而 `login` 方法在查询时使用了 `trim()` 后的用户名，导致无法匹配。修复方案是在 `register` 方法中也对用户名进行 `trim()`，确保存储和查询的Key一致。”

### 及格总结
“用户名有空格导致的bug。需要在注册时trim一下。”
