# 系统设计与技术架构

## 一、核心功能模块

### 1. 大脑 - 多角色 Agent 中枢
基于 Gemma 4 31B Dense 模型构建，核心能力：
- **`create_mission(role_id, difficulty)`**: 动态生成符合职业与难度的具体任务、Mock数据及交付要求。
- **`evaluate_submission(submission_text, mission_description)`**: 评估用户提交物，生成结构化反馈与技能经验值。
- **`act_as_role(role_name, user_input)`**: 在不同AI角色（严厉主管、热心同事、挑剔客户）间切换，进行情境对话。

### 2. 骨架 - 职业剧本与知识库 (RAG)
- **内容**：为每个核心职业设计3-4个阶梯式任务剧本。
- **知识库**：为每个任务配套高质量的Markdown文档库（概念、代码、最佳实践），存放于本地 `docs/knowledge_base/` 目录下。
- **存储与检索引擎**：
  - **关系型存储**：采用轻量级关系数据库 SQLite（支持无缝扩展至 PostgreSQL），用于持久化玩家角色状态、任务进行记录以及技能树等级和经验值。
  - **向量检索引擎**：采用轻量级向量数据库 ChromaDB（或 FAISS），将本地 Markdown 教程进行文本分块 (Chunking) 与向量化嵌入 (Embeddings)，在用户遇到瓶颈时通过语义检索匹配最相关的技术知识，实现精准学习资源推荐。

### 3. 血肉 - 游戏化前端
- **风格**：经典2D像素艺术风格，重信息、轻拟真。
- **核心界面**：
  - **职业选择大厅**：世界地图，选择进入不同职业岛屿。
  - **职业中枢**：集成任务面板、技能树、资源书架、AI导师形象。
  - **任务交互**：包含任务详情、进行中状态、提交与AI反馈展示。

### 4. 灵魂 - 反馈与成长系统
- **费曼挑战**：AI随机发起深度理解性质疑。
- **技能经验值**：AI评估后输出结构化数据（如 `{"skill_data_cleaning": +10}`），驱动前端技能树可视化成长。

## 二、世界观与交互设计 (像素风)
- **核心视觉**：采用限制色板的2D像素艺术，所有界面元素由纯色块、1像素边框、像素图标和点阵字体构成。
- **关键界面状态**：定义了“职业大厅”、“职业中枢”、“任务弹窗”、“提交反馈”等界面的像素级交互状态与反馈动画。
- **设计目标**：在资源有限下，通过清晰的视觉逻辑和动效，营造沉浸感。
